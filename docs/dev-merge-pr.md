# Dev Integration Summary

## Summary

`dev` contains the current vertical slice for SoundCluster:

- metadata search
- lyrics lookup
- Gemini analysis
- MySQL cache
- SSE progress stream
- 3D emotion-space rendering
- relation highlighting
- share URL persistence

`dev` has been fast-forward merged into `main`.

## Implemented Scope

### Foundation

- Node, pnpm, TypeScript, Vite, Express setup.
- MySQL connection pool.
- Gemini SDK setup.
- Shared constants and shared types.

### Analysis Pipeline

- `/api/analyze/stream` SSE endpoint.
- Cache key normalization by title and artist.
- MySQL-backed analysis cache.
- Gemini response validation for five `0.0 ~ 1.0` emotion values.
- SSE phases: `fetching`, `analyzing`, `done`, `failed`.

### Metadata Pipeline

- `/api/itunes/search` backend proxy.
- iTunes metadata normalized into title, artist, album image, iTunes URL, and track id.
- `/api/lyrics/search` backend proxy.
- LRCLIB lookup uses lyrics when available and falls back safely when lyrics are missing.

### Frontend Rendering

- React HUD for search, search results, emotions, selected track, debug response, and share action.
- R3F canvas with starfield, XYZ axis lines, track nodes, relation lines, and OrbitControls.
- Axis toggles choose which emotion dimensions participate in projection.
- Song emotion values are generated by analysis and are not user-editable.
- Selected track, nearest track, and farthest track are emphasized by scale.

### Sharing

- `/api/share-snapshots` stores validated snapshots in MySQL.
- `nanoid` share ids are reused for identical snapshot hashes.
- Shared URL loads snapshot by id and recomputes projection on the frontend.

### Tests

- Analysis cache key normalization.
- Gemini response range validation.
- SSE fixture order.
- Stable 5D to 3D mapping.
- Share snapshot encoding and validation.
- Track relation calculation.
